# 신재성 파트 실제 코드 기준 기능 설명

이 문서는 기능 하나를 설명할 때 `VO -> Mapper -> XML(SQL) -> ServiceImpl -> Controller -> HTML/JS` 흐름으로 읽기 위한 자료이다.

CSS 설명은 제외하고, 실제 동작을 만드는 Java / MyBatis XML / Thymeleaf / JS 중심으로 정리한다.

---

## 1. 코드값 목록 / 검색

### VO

파일: `src/main/java/com/pixcel/app/codevalue/service/CodeValueVO.java`

```java
private String settingCodeId;
private String userId;
private String settingGroupName;
private List<String> settingGroupNameList;
private String settingName;
private String defaultYn;
private List<String> defaultYnList;
private String useYn;
```

설명:
- `settingGroupNameList`, `defaultYnList`는 체크박스 다중 검색을 받기 위해 추가한 필드다.
- 기존 단일 검색값 `settingGroupName`, `defaultYn`도 남겨서 예전 방식과 호환되게 했다.

### XML(SQL)

파일: `src/main/resources/mapper/codevalue/codevalue-map.xml`

```xml
<select id="selectCodeValueSearchList"
    parameterType="CodeValueVO" resultType="CodeValueVO">
    select
    setting_code_id,
    user_id,
    setting_group_name,
    setting_name,
    default_yn,
    use_yn
    from setting_code
    where user_id = #{userId}
    and setting_group_name in ('g001', 'g002', 'g003')

    <if test="settingGroupNameList != null and settingGroupNameList.size() > 0">
        and setting_group_name in
        <foreach collection="settingGroupNameList" item="settingGroupName"
            open="(" separator="," close=")">
            #{settingGroupName}
        </foreach>
    </if>

    <if test="settingName != null and settingName != ''">
        and upper(setting_name) like '%' || upper(#{settingName}) || '%'
    </if>

    <if test="defaultYnList != null and defaultYnList.size() > 0">
        and default_yn in
        <foreach collection="defaultYnList" item="defaultYn"
            open="(" separator="," close=")">
            #{defaultYn}
        </foreach>
    </if>

    order by setting_group_name, setting_code_id
</select>
```

설명:
- `where user_id = #{userId}`로 로그인 사용자의 코드값만 조회한다.
- `setting_group_name in ('g001', 'g002', 'g003')`로 코드값 관리 화면에서 다루는 그룹만 제한한다.
- `<if>`는 검색조건이 있을 때만 SQL 조건을 붙이는 MyBatis 동적 SQL이다.
- `<foreach>`는 체크박스 여러 개 선택 시 `in (?, ?, ?)` 형태로 만들어준다.

### ServiceImpl

파일: `src/main/java/com/pixcel/app/codevalue/service/impl/CodeValueServiceImpl.java`

```java
@Override
public Map<String, List<CodeValueVO>> getCodeValueSearchGroupMap(CodeValueVO searchVO) {

    String selectedGroupName = searchVO.getSettingGroupName();
    List<String> selectedGroupNameList = searchVO.getSettingGroupNameList();
    List<CodeValueVO> codeValueList;

    searchVO.setSettingGroupName(null);
    searchVO.setSettingGroupNameList(null);

    try {
        codeValueList = getCodeValueSearchList(searchVO);
    } finally {
        searchVO.setSettingGroupName(selectedGroupName);
        searchVO.setSettingGroupNameList(selectedGroupNameList);
    }

    Map<String, List<CodeValueVO>> groupMap = new LinkedHashMap<>();

    groupMap.put(GROUP_G001, filterByGroup(codeValueList, GROUP_G001));
    groupMap.put(GROUP_G002, filterByGroup(codeValueList, GROUP_G002));
    groupMap.put(GROUP_G003, filterByGroup(codeValueList, GROUP_G003));

    return groupMap;
}
```

설명:
- 화면은 그룹 카드 3개를 항상 보여줘야 한다.
- 그래서 그룹 필터를 실제 SQL에 바로 걸면 선택된 그룹만 화면에 남게 된다.
- 중간에 `settingGroupName`, `settingGroupNameList`를 잠깐 null로 만든 이유는 전체 그룹 데이터를 조회한 뒤, Java에서 그룹별로 다시 나누기 위해서다.
- 그 결과 선택된 그룹만 사라지는 게 아니라, 전체 그룹은 유지되고 선택된 그룹은 CSS로 강조할 수 있다.

```java
private void normalizeSearchCondition(CodeValueVO searchVO) {
    searchVO.setSettingGroupNameList(normalizeSearchList(searchVO.getSettingGroupNameList(),
            searchVO.getSettingGroupName()));
    searchVO.setDefaultYnList(normalizeSearchList(searchVO.getDefaultYnList(), searchVO.getDefaultYn()));
}
```

설명:
- checkbox 방식은 `List<String>`으로 값이 들어온다.
- 예전 select 방식은 단일 문자열로 들어왔다.
- 둘 중 어떤 방식으로 들어와도 동일하게 처리하기 위해 리스트로 정규화한다.

```javascript
function updateFilterSummary(dropdown) {
    var checkedLabels = Array.prototype.slice.call(
        dropdown.querySelectorAll('input[type="checkbox"]:checked, input[type="radio"]:checked'))
        .map(function(input) {
            var label = input.closest('.filter-check-item');
            return label ? label.innerText.trim() : '';
        })
        .filter(function(text) {
            return text.length > 0;
        });

    var summaryText = dropdown.querySelector('.filter-selected-text');
    var placeholder = dropdown.getAttribute('data-placeholder') || '전체';

    if (checkedLabels.length === 0) {
        summaryText.textContent = placeholder;
        dropdown.classList.remove('has-selected');
        return;
    }

    summaryText.textContent = checkedLabels.join(', ');
    dropdown.classList.add('has-selected');
}
```

설명:
- 체크된 항목의 라벨 텍스트를 읽어서 드롭다운 요약 영역에 표시한다.
- 검색은 서버가 처리하지만, 선택한 값이 화면 필터에 보이는 부분은 JS가 담당한다.

## 2. 코드값 생성

### XML(SQL)

```xml
<insert id="insertCodeValue" parameterType="CodeValueVO">

    <selectKey keyProperty="settingCodeId" resultType="string"
        order="BEFORE">
        select
        'SETTING_CODE_'
        || to_char(sysdate, 'YYMM')
        || '_'
        || lpad(
        nvl(max(to_number(substr(setting_code_id, -4))), 0) + 1,
        4,
        '0'
        )
        from setting_code
        where substr(setting_code_id, 1, 18) =
        'SETTING_CODE_' || to_char(sysdate, 'YYMM') || '_'
    </selectKey>

    insert into setting_code (
    setting_code_id,
    user_id,
    setting_group_name,
    setting_name,
    default_yn
    ) values (
    #{settingCodeId},
    #{userId},
    #{settingGroupName},
    #{settingName},
    #{defaultYn}
    )
</insert>
```

설명:
- 별도 시퀀스 테이블을 쓰지 않고 MyBatis `selectKey`로 ID를 생성한다.
- `SETTING_CODE_2606_0001` 같은 형식이 된다.
- `use_yn`은 insert에서 제외했다. 이 화면의 관리 대상이 아니고 DB 기본값을 사용한다.

### ServiceImpl

```java
@Override
@Transactional
public void createCodeValue(CodeValueVO codeValue, String userId) {
    validateUserId(userId);
    validateCreateCodeValue(codeValue);

    codeValue.setUserId(userId);

    if (codeValue.getDefaultYn() == null || codeValue.getDefaultYn().trim().isEmpty()) {
        codeValue.setDefaultYn("N");
    }

    int duplicateCount = codeValueMapper.countDuplicateSettingName(
            userId,
            codeValue.getSettingGroupName(),
            codeValue.getSettingName()
    );

    if (duplicateCount > 0) {
        throw new IllegalArgumentException("이미 존재하는 코드값 이름입니다.");
    }

    if ("Y".equals(codeValue.getDefaultYn())) {
        int defaultCount = codeValueMapper.countDefaultCodeValue(userId, codeValue.getSettingGroupName());

        if (defaultCount > 0) {
            throw new IllegalArgumentException("해당 그룹에는 이미 기본값이 존재합니다.");
        }
    }

    codeValueMapper.insertCodeValue(codeValue);
}
```

설명:
- `@Transactional`은 중복검사 후 insert까지 하나의 작업으로 묶기 위해 붙였다.
- 같은 사용자, 같은 그룹 안에서 이름 중복을 막는다.
- 기본값은 그룹당 하나만 허용한다.

### Controller

```java
@PostMapping("/codevalue/create")
public String codeValueCreate(@CookieValue(value = "userId", required = false) String userId,
                              CodeValueVO codeValue,
                              RedirectAttributes redirectAttributes) {

    String loginUserId = getLoginUserId(userId);

    try {
        codeValueService.createCodeValue(codeValue, loginUserId);
        redirectAttributes.addFlashAttribute("message", "코드값이 등록되었습니다.");

        return "redirect:/codevalue/list";

    } catch (IllegalArgumentException e) {
        redirectAttributes.addFlashAttribute("errorMessage", e.getMessage());
        redirectAttributes.addFlashAttribute("codeValue", codeValue);

        return "redirect:/codevalue/create?settingGroupName=" + codeValue.getSettingGroupName();
    }
}
```

설명:
- 성공하면 목록으로 이동한다.
- 실패하면 입력값을 flashAttribute로 다시 넘겨서 생성 화면에 유지한다.

## 4. 일감상태 생성 / 수정 / 삭제

### 생성 SQL

```xml
<insert id="insertIssueStatus" parameterType="IssueStatusVO">
    <selectKey keyProperty="issueStatusId" resultType="string"
        order="BEFORE">
        select
        'ISSUE_STATUS_'
        || to_char(sysdate, 'YYMM')
        || '_'
        || lpad(
        nvl(max(to_number(substr(issue_status_id, -4))), 0) + 1,
        4,
        '0'
        )
        from issue_status
        where substr(issue_status_id, 1, 18) =
        'ISSUE_STATUS_' || to_char(sysdate, 'YYMM') || '_'
    </selectKey>

    insert into issue_status (
    issue_status_id,
    user_id,
    status_name,
    status_answer,
    closed_yn
    ) values (
    #{issueStatusId},
    #{userId},
    #{statusName},
    #{statusAnswer},
    #{closedYn}
    )
</insert>
```

### ServiceImpl

```java
@Override
@Transactional
public void createIssueStatus(IssueStatusVO issueStatus, String userId) {
    validateUserId(userId);

    issueStatus.setUserId(userId);

    validateIssueStatus(issueStatus);

    int duplicateCount = issueStatusMapper.countDuplicateStatusName(issueStatus.getUserId(),
            issueStatus.getStatusName(), null);

    if (duplicateCount > 0) {
        throw new IllegalArgumentException("이미 사용 중인 상태명입니다.");
    }

    issueStatusMapper.insertIssueStatus(issueStatus);
}
```

설명:
- 상태명은 사용자별로 중복될 수 없게 했다.
- `closedYn` 값이 없으면 `validateIssueStatus()`에서 기본값 `N`으로 세팅한다.

### 삭제 SQL

```xml
<select id="countUsedIssueStatus" resultType="int">
    select count(*)
    from (
    select issue_status_id
    from issue
    where issue_status_id = #{issueStatusId}

    union all

    select initial_status_id
    from issue_type
    where initial_status_id = #{issueStatusId}

    union all

    select from_status_id
    from workflow_transition
    where from_status_id = #{issueStatusId}

    union all

    select to_status_id
    from workflow_transition
    where to_status_id = #{issueStatusId}
    )
</select>
```

설명:
- 일감 상태는 `issue`, `issue_type`, `workflow_transition`에서 모두 쓰인다.
- 그래서 삭제 전 연결 여부를 반드시 확인한다.

### 삭제 ServiceImpl

```java
@Override
@Transactional
public void removeIssueStatus(String issueStatusId, String userId) {
    validateUserId(userId);

    IssueStatusVO issueStatus = issueStatusMapper.selectIssueStatusDetail(issueStatusId, userId);

    if (issueStatus == null) {
        throw new IllegalArgumentException("존재하지 않는 일감 상태입니다.");
    }

    int usedCount = issueStatusMapper.countUsedIssueStatus(issueStatusId);

    if (usedCount > 0) {
        throw new IllegalArgumentException("이미 일감, 일감유형 또는 업무흐름에서 사용 중인 상태이므로 삭제할 수 없습니다.");
    }

    issueStatusMapper.deleteIssueStatus(issueStatusId, userId);
}
```

설명:
- 사용 중인 상태를 삭제하면 일감/유형/업무흐름이 깨진다.
- 그래서 삭제보다 참조 검사 로직이 더 중요하다.

### XML(SQL)

파일: `src/main/resources/mapper/issuetype/issuetype-map.xml`

```xml
<select id="selectIssueTypeListPageRows" parameterType="IssueTypeVO"
    resultType="IssueTypeVO">
    select *
    from (
    select
    'ISSUE_TYPE' as option_group,

        and upper(it.issue_type_name) like '%' ||
        upper(#{issueTypeName}) || '%'
    </if>

    <if test="initialStatusIdList != null and initialStatusIdList.size() > 0">
        and it.initial_status_id in
        <foreach collection="initialStatusIdList" item="initialStatusId"
            open="(" separator="," close=")">
            #{initialStatusId}
        </foreach>
    </if>

    union all

    select
    'ISSUE_STATUS' as option_group,
    null as issue_type_id,
    null as initial_status_id,
    null as issue_type_name,
    null as roadmap_yn,
    null as description,
    null as created_at,
    null as updated_at,
    user_id,
    null as initial_status_name,
    issue_status_id,
    status_name,
    null as project_id,
    null as project_name,
    2 as option_sort,
    row_number() over (order by issue_status_id asc) as row_sort
    from issue_status
    where user_id = #{userId}
    )
    order by option_sort, row_sort
</select>
```

설명:

- 각 row에 `option_group`을 붙여 서비스에서 분리한다.
- CLOB 설명 검색은 `dbms_lob.substr()`로 잘라서 검색한다.

## 6. 일감유형 생성 / 복사

### VO 핵심 필드

```java

@Transactional
@Override
public void saveWorkflowTransition(WorkflowVO workflowVO, String userId) {

    validateUserId(userId);

    workflowVO.setUserId(userId);
    workflowVO.setCreatedBy(userId);

    validateRequired(workflowVO.getIssueTypeId(), "일감유형");
    validateRequired(workflowVO.getRoleId(), "역할");
    validateApplyTargetCode(workflowVO.getApplyTargetCode());

    validateIssueTypeOwner(workflowVO.getIssueTypeId(), userId);
    validateRoleOwner(workflowVO.getRoleId(), userId);

    List<WorkflowVO> transitionList = parseTransitionKeyList(workflowVO);
    validateStatusOwner(transitionList, userId);

    String requestedApplyTargetCode = workflowVO.getApplyTargetCode();
    List<String> saveApplyTargetCodeList = getSaveApplyTargetCodeList(requestedApplyTargetCode);
    List<WorkflowVO> saveTransitionList = new ArrayList<>();

    for (String applyTargetCode : saveApplyTargetCodeList) {
        workflowVO.setApplyTargetCode(applyTargetCode);
        workflowMapper.deleteWorkflowTransition(workflowVO);

        for (WorkflowVO transition : transitionList) {
            saveTransitionList.add(createSaveTransition(workflowVO, transition, applyTargetCode, userId));
        }
    }

    if (!saveTransitionList.isEmpty()) {
        workflowMapper.insertWorkflowTransitionList(saveTransitionList);
    }

    workflowVO.setApplyTargetCode(requestedApplyTargetCode);
}
```

설명:
- 기존 전환을 수정하는 방식이 아니라, 해당 조건의 전환을 삭제 후 다시 저장한다.
- 체크박스 매트릭스 화면에서는 이 방식이 단순하고 안전하다.
- `j001 기본`을 저장하면 `getSaveApplyTargetCodeList()`에 의해 `j001`, `j002`, `j003`까지 같이 저장된다.

### parseTransitionKeyList 코드

```java
private List<WorkflowVO> parseTransitionKeyList(WorkflowVO workflowVO) {

    List<WorkflowVO> transitionList = new ArrayList<>();

    List<String> transitionKeyList = workflowVO.getTransitionKeyList();

    if (transitionKeyList == null || transitionKeyList.isEmpty()) {
        return transitionList;
    }

    for (String transitionKey : transitionKeyList) {

        validateRequired(transitionKey, "상태전환 값");

        String[] statusIds = transitionKey.split("\\|");

        if (statusIds.length != 2) {
            throw new IllegalArgumentException("상태전환 값 형식이 올바르지 않습니다.");
        }

        String fromStatusId = statusIds[0];
        String toStatusId = statusIds[1];

        if (fromStatusId.equals(toStatusId)) {
            throw new IllegalArgumentException("같은 상태로는 전환할 수 없습니다.");
        }

        WorkflowVO transition = new WorkflowVO();
        transition.setFromStatusId(fromStatusId);
        transition.setToStatusId(toStatusId);

        transitionList.add(transition);
    }

    return transitionList;
}
```

설명:
- 화면에서 넘어온 문자열을 실제 DB 컬럼 구조로 변환한다.
- 같은 상태로 이동하는 전환은 저장하지 못하게 막는다.

### 수정 ServiceImpl

```java
@Override
@Transactional
public void updateIssue(IssuesVO issue, String userId) {
    validateUserId(userId);
    validateBasicIssueForUpdate(issue);
    validateProjectAccess(issue.getProjectId(), userId);

    if (!hasProjectPermission(issue.getProjectId(), userId, PERMISSION_ISSUE_UPDATE_CODE,
            PERMISSION_ISSUE_UPDATE_NAME)) {
        throw new IllegalArgumentException("일감 수정 권한이 없습니다.");
    }

    IssuesVO currentIssue = issuesMapper.selectIssueForUpdate(issue.getProjectId(), issue.getIssueId());

    if (currentIssue == null) {
        throw new IllegalArgumentException("수정할 일감이 없습니다.");
    }

    normalizeIssueForUpdate(issue, currentIssue);
    validateLength(issue);
    validateProgressRate(issue);
    validatePriority(issue.getProjectId(), issue.getSettingCodeId());
    validateDynamicFieldsForUpdate(issue);
    validateStatusTransition(currentIssue, issue, userId);

    int updateCount = issuesMapper.updateIssue(issue);

    if (updateCount == 0) {
        throw new IllegalArgumentException("수정할 일감이 없습니다.");
    }

    recordUpdateHistory(currentIssue, issue, userId);
}
```

설명:
- 수정 전 기존 일감을 `selectIssueForUpdate`로 가져온다.

- 수정 후 변경 이력을 기록한다.

## 12. 일감 삭제

### ServiceImpl

```java
@Override
@Transactional
public void deleteIssue(String projectId, String issueId, String userId) {
    validateProjectAccess(projectId, userId);

    if (isBlank(issueId)) {
        throw new IllegalArgumentException("삭제할 일감 정보가 없습니다.");
    }

    if (!hasProjectPermission(projectId, userId, PERMISSION_ISSUE_DELETE_CODE, PERMISSION_ISSUE_DELETE_NAME)) {
        throw new IllegalArgumentException("일감 삭제 권한이 없습니다.");
    }

    if (issuesMapper.countChildIssue(projectId, issueId) > 0) {
        throw new IllegalArgumentException("다른 일감의 상위일감으로 사용 중인 일감은 삭제할 수 없습니다.");
    }

    try {
        issuesMapper.deleteIssueHistory(issueId);
        int deleteCount = issuesMapper.deleteIssue(projectId, issueId);

        if (deleteCount == 0) {
            throw new IllegalArgumentException("삭제할 일감이 없습니다.");
        }
    } catch (DataIntegrityViolationException e) {
        throw new IllegalArgumentException("연결된 데이터가 있는 일감은 삭제할 수 없습니다.");
    }
}
```

설명:
- 삭제 권한 `p010`을 확인한다.
- 다른 일감의 상위일감이면 삭제를 막는다.
- 이력이 연결되어 있으므로 `issue_history`를 먼저 삭제하고 `issue`를 삭제한다.

### XML(SQL)

```xml
<select id="countChildIssue" resultType="int">
    select count(*)
    from issue
    where project_id = #{projectId}
    and parent_issue_id = #{issueId}
</select>

<delete id="deleteIssueHistory">
    delete from issue_history
    where issue_id = #{issueId}
</delete>

<delete id="deleteIssue">
    delete from issue
    where project_id = #{projectId}
    and issue_id = #{issueId}
</delete>
```

설명:
- 삭제 순서는 `하위일감 확인 -> 이력 삭제 -> 일감 삭제`다.
